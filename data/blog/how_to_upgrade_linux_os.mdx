## 🚀 본론: Linux OS 업그레이드 절차

### ✅ Step 1 - Linux OS 업그레이드 절차

```ex 💻
# 1. Integration Service(IS)를 안전 모드로 설정 후 비활성화

- Admin Console 접속 → Integration Services → 비활성화

# 2. 다음 항목들의 백업을 반드시 수행

- Domain, Repository 백업
- nodemeta.xml, keystore, truststore, sitekeys, ODBC.ini, .bash_profile 백업

# 3. 모든 노드에서 Informatica 서비스를 중지

# 4. OS 업그레이드 수행

# 5. 업그레이드 후 확인 사항

- ODBC.ini 파일을 점검하고, 모든 노드에서 환경 변수 값이 변경되지 않았는지 확인
- INFA 관련 환경 변수 경로는 업그레이드 전과 동일해야 함

# 6. 서비스 재기동 후 Admin Console 정상 여부 확인

# ETC

- 안전 모드에서 IS를 실행한 후 관리자 계정으로 클라이언트 도구에 로그인하여 테스트 진행

Oracle을 소스로, Flat-file을 타겟으로 하는 샘플 워크플로우 생성 후 데이터 로딩
동일한 파일을 사용해 유사한 구조의 Oracle 테이블에 데이터 로딩
SQL Server에 대한 일대일(one-to-one) 로드 테스트
사용 중인 다양한 연결(Connection)에 대한 연결 테스트 수행
ISQL, sqlplus 등을 사용하여 연결 테스트 진행
```

## 📋 결론

- 위와 같은 절차대로 수행하면 안전하게 Linux OS 업그레이드가 가능합니다.
